Mail.js

What

Mail.js is an open-source JavaScript library designed to help render email.

Note that Mail.js is not a mail transport -- Mail.js works with mail transport packages like Nodemailer.

Why

Email is notoriously hard to format for correctly. Here are some examples of the problems:
  • Most web mail readers support body <style> except gmail
  • Formatting both text and HTML email messages without message duplication can be painful

How

Mail.js uses a simple template system which allows you to simply craft your emails while making it easy to generate both text and HTML messages from the same source. It also bakes in email boilerplate like HTML Email Boilerplate behind the scenes to solve many of the email rendering idiosyncrasies across email renderers.

Install

npm install mailjs --save

Mail.js is hosted on Github.

API

Methods

mailjs.config() configures Mail.js options:

mailjs.config({
  binds: An object containing common bind definitions.
  templates: An object containing common template definitions.
});

mailjs.render() formats email:

mailjs.render({
  src: A string containing the template to render.
  binds: An object containing bind definitions that will be available while processing this render.
  templates: An object containing template definitions that will be available while processing this render.
  html: A boolean indicating whether HTML (true) or text (false) should be rendered.
});

Rendering Emails

Mail.js uses binds and templates to render email.

Bindings and templates are late bound -- this means you can provide a definition for a binding or template after it has been defined.

Bindings and templates can also recursively reference other bindings and templates. (Be careful not to cause infinite recursive cycles.)

Binds

Binds are simple variables bindings and are essentially lightweight templates. Their syntax is similar to string interpolation in EcmaScript 6. 

src: 'Hello $firstName ${lastName},'
Bind Usage Example

Mail.js' builtin email boilerplate has several bind hooks that you can optionally override:

  • $title - Used as a title of the email. Optional.
  • $headBlock - Extra styles to be placed in the head style block.

Templates

Templates are more advanced bindings. And provide a number of options: 

template options: {
  html: A string containing the HTML to display for this template when rendering HTML.
  text: A string containing the text to display for this template when rendering text.
  src: A string that will work for both HTML and text rendering.  If specifing html and text, do not specify src.
  htmlClose: HTML string for closing template tags.
  textClose: Text string for closing template tags.
  srcClose: A string that will work for both HTML and text rendering for closing template tags.
  binds: An object containing bind definitions that will be added to the scope when processing this template.
  templates: An object containing template definitions that will be added to the scope when processing this template.
}
Template Options

It is often possible to use src-style tags because the template string can contain references to other templates that abstract the differences between HTML and text. Essentially if you abstract the differences between HTML and text into simple templates then your composite directives will tend to be more reusable across HTML and text.

Templates syntax is similar to HTML's use of angular brackets, but with square brackets instead: 

src: '[a href="$companyUrl"]our url[/a]'
Template Usage Example

A shortcut to assign a parameter with the same name as the template exists. The following two are equivalent: 

src: '[email email="$companyUrl" label="a label"]'

The following is a shortcut of the preceding:

src: '[email="$companyUrl" label="a label"]'

Example

Use the config to define common bindings and templates:

mailjs.config({
  binds: {
    company: 'ACME Corp'
  },
  templates: {
    page: {
      html:  '<div style="color:#222;font-weight:bold;">ACME Corp<div>',
      text:  'ACME Corp\\n---------\\n\\n',
      htmlClose:  '<div style="color:#ccc;font-size:12px;">Copyright 2014<div>',
      textClose:  'Copyright 2014'
    },
    email: {
      html: '<a href="mailto:$email">$email',
      text: '$email'
    }
  }
});
Mail.js config example

A simple render example:

mailjs.render({
  src: '[page]Thank you for your order.  Contact us at [email=$companyEmail][/page]',
  binds: {
    title: '$company -- Order Complete'
  },
  html: true
}
